GlossaryStable

📚Common terms used throughout this playbook87 terms
Use this glossary to align product, engineering, security, data, and operations teams on the vocabulary used in agentic AI production systems.
TL;DR before this long table
  • Use this table to compare tradeoffs quickly; start with the default or mature option before adopting specialized alternatives.
  • Check the freshness badge or review date for market/vendor claims before making a production decision.
  • For high-risk systems, require observability, access control, evaluation coverage, and rollback before launch.
TermMeaning
A2A ProtocolAgent-to-agent communication pattern for coordinating agents across systems, vendors, or execution boundaries.
A2UIAgent-to-UI pattern for generating or controlling user interface components from agent state and events.
ACL-Aware RetrievalRetrieval that filters documents, chunks, and citations according to user, tenant, role, group, or document-level permissions before content reaches the model.
AgentA system that can interpret a goal, reason over context, choose actions, call tools, observe results, and continue until a stop condition or handoff.
Agent FrameworkThe abstraction layer for building agents: model calls, prompt templates, tool binding, structured outputs, message formatting, retrieval helpers, and reusable agent patterns.
Agent HarnessThe outer session or interaction shell that starts an agent run, loads project context, manages the workspace or channel, and returns output to a user or calling system.
Agent RuntimeThe execution engine that manages state over time: checkpoints, retries, graph transitions, fan-out/fan-in, human interrupts, durable execution, and long-running workflow control.
Agentic WorkflowA multi-step workflow where an AI system performs planning, tool use, retrieval, validation, and iteration rather than producing a single response.
AG-UIAn agent-to-user-interface event protocol pattern that lets agents stream state, actions, intermediate outputs, and UI updates to applications.
Approval GateA required human, policy, or automated approval step before a high-risk action, irreversible side effect, or release promotion can continue.
ArtifactA durable output produced by an AI system, such as a report, code diff, plan, summary, data file, chart, or generated document.
Audit LogAn immutable or tamper-evident record of user actions, model calls, tool calls, approvals, policy decisions, and system events used for governance and incident review.
Autonomy LadderA progression from passive suggestions to tool execution with approval, supervised automation, and finally bounded autonomous operation.
Build-vs-BuyA decision framework for choosing whether to adopt an existing managed/tooling layer or build a custom internal component.
Canary ReleaseA controlled rollout where a new model, prompt, tool, or workflow is exposed to a small slice of traffic before broader promotion.
Capability BoundaryThe line where an agent gains access to an external action, data source, resource, or workflow; this boundary should be typed, permissioned, logged, and policy-controlled.
Capability DiscoveryA mechanism by which a host or agent learns what tools, resources, prompts, and schemas are available from an interface such as MCP.
CheckpointerA persistence mechanism that saves agent state at key steps so conversations or workflows can resume, branch, audit, or roll back.
ChunkA segment of a source document prepared for retrieval, usually with metadata such as source, section, tenant, timestamp, and permissions.
Context + Capability InterfaceThe boundary where agents access external capabilities and context through typed tools, resources, prompts, retrieval connectors, OpenAPI schemas, function tools, or MCP servers.
Context EngineeringThe design of what information enters the model context, including instructions, retrieved evidence, memory, tool results, policy, and conversation history.
Context WindowThe maximum amount of input and output text a model can consider in one request, usually measured in tokens.
CopilotKitReact-first framework for embedding AI copilots and agentic workflows inside production applications.
Cost RouterA routing layer that selects cheaper or more capable models depending on task difficulty, latency needs, confidence, and budget constraints.
Cross-Cutting PlaneA concern that spans multiple stack layers rather than living in one layer, such as observability, evaluation, security, governance, privacy, cost, and policy.
DAG / WorkflowA directed acyclic graph that defines deterministic task dependencies; useful when an agentic process should be constrained into known steps.
Data LineageThe traceable path of data from source systems through ingestion, transformation, retrieval, prompts, outputs, and stored artifacts.
DatabricksEnterprise lakehouse and AI platform for governed analytical data, Delta Lake/Unity Catalog access control, feature and ML pipelines, vector search, MLflow evaluation, and model or agent deployment. In this playbook it is a data + governance + AI platform that agents can call into, not a replacement for an orchestration framework.
Durable ExecutionExecution that persists workflow state so a run can survive process crashes, restarts, retries, deployments, or long human approval waits.
EmbeddingA numerical vector representation of text, image, or other content used for semantic search, clustering, deduplication, or similarity matching.
Eval HarnessThe system that runs test cases against prompts, models, retrieval pipelines, tools, and agent trajectories, then records pass/fail and quality metrics.
Execution SubstrateThe actual environment or external system where side effects occur: containers, sandboxes, browsers, Python kernels, filesystems, databases, queues, storage, or SaaS APIs.
Fan-out / Fan-inA runtime pattern where work is split across parallel branches or agents, then joined back into a single downstream step or synthesis phase.
Freshness CheckA validation step that verifies whether retrieved content, source claims, model capabilities, or vendor details are current enough for the decision being made.
Generative UIUmbrella pattern where agents produce interface events, component descriptions, or validated UI schemas that client applications render safely.
Golden DatasetA curated set of test inputs with expected answers, source evidence, rubrics, or tool-call expectations used as a release gate.
GraphRAGA retrieval pattern that combines graph relationships with semantic retrieval so answers can use entity links, dependencies, and multi-hop context.
GroundednessThe degree to which an answer is supported by retrieved or cited source evidence rather than unsupported model inference.
GuardrailA control that constrains model behavior, tool use, outputs, inputs, or policy compliance through rules, classifiers, validators, or human review.
HITLHuman-in-the-loop. A design pattern where humans review, approve, correct, or take over high-risk, low-confidence, or policy-sensitive steps.
IdempotencyA property of an operation where repeated execution has the same effect as one execution, which is critical for safe retries and tool calls.
Inner Behavioral LoopThe repeated decision pattern inside an agent run, such as Think → Act → Observe, ReAct, plan-and-execute, reflection, or evaluator-optimizer loops.
Instruction HierarchyThe ordering of authority among system instructions, developer instructions, user requests, retrieved content, memory, and tool outputs.
InterruptA runtime primitive that pauses execution, serializes state, and waits for a human approval, correction, external event, or policy decision before continuing.
MCPModel Context Protocol; a standard interface pattern for connecting agents or model applications to tools, data sources, prompts, resources, and external systems.
MCP AppsMCP-native app-surface pattern that packages tools, resources, and workflows into user-facing experiences.
MCP-UIPattern for returning interactive UI surfaces through MCP tool or resource contexts.
MemoryStored user, task, or system information that can be recalled across turns or sessions, often separated into short-term, long-term, episodic, or semantic memory.
Model FallbackA model-specific fallback path that routes to another model when the primary model fails, times out, exceeds budget, violates policy, or scores below confidence thresholds.
Model Gateway / AI GatewayA model-access control layer between applications, agents, and model providers. It centralizes provider abstraction, routing, retries, fallbacks, virtual keys, budgets, spend tracking, logging, caching, and policy hooks. Examples include LiteLLM, Portkey, OpenRouter, Kong AI Gateway, and Cloudflare AI Gateway.
LiteLLMAn open-source AI gateway and Python SDK that provides a unified OpenAI-compatible interface across many model providers. In production, it is commonly used as a centralized proxy for model routing, virtual keys, budgets, spend tracking, fallbacks, caching, and logging.
Model RegressionA quality, safety, cost, or latency drop introduced by changing the model, prompt, retrieval pipeline, tool schema, or orchestration logic.
Model RouterA routing layer that chooses a model based on task type, cost, latency, context size, modality, risk, region, or required capability.
Model RoutingThe process of selecting which model or provider should handle a request based on task type, cost, latency, availability, policy, data boundary, or fallback rules.
Multi-Agent OrchestrationCoordination of multiple agents with specialized roles, shared state, message passing, task delegation, and conflict resolution.
Observability TraceA structured record of a model or agent run, including prompts, outputs, retrieval hits, tool calls, latency, costs, errors, and policy events.
Open-JSON-UIDeclarative JSON contract for generated UI components, layouts, bindings, and actions.
OPAOpen Policy Agent; a policy engine commonly used for authorization, entitlements, and governance decisions.
OpenTelemetry / OTELA vendor-neutral telemetry standard for traces, metrics, and logs that helps connect agent traces with infrastructure and application observability.
PIIPersonally identifiable information; data that can identify, contact, locate, or distinguish a person, directly or in combination with other data.
Policy-as-CodeEncoding access, safety, privacy, approval, and governance rules in versioned, testable policy logic rather than only in prose or manual process.
PostconditionA condition that must be true after a tool call or workflow step, such as record created, email sent, balance unchanged, or approval logged.
Product / Control PlaneThe layer above the agent harness that owns product workflow, user/org permissions, workflow catalog, approval queues, quotas, audit posture, launch gates, and governance controls.
Prompt InjectionAn attack where malicious or untrusted content tries to override instructions, reveal secrets, misuse tools, or alter system behavior.
Prompt Resource / Prompt TemplateA reusable prompt, instruction block, or template exposed by a framework, host, or MCP server for consistent agent behavior.
PromptFooAn evaluation and red-team testing tool used to compare prompts, models, outputs, and policy behaviors across test cases.
RAGRetrieval-augmented generation. A pattern where the system retrieves external knowledge and supplies it to the model to improve accuracy and provenance.
RBACRole-based access control. A permission model where capabilities are assigned to roles and users inherit the permissions of their roles.
Red-Team SuiteA set of adversarial tests designed to probe unsafe behavior, security failures, jailbreaks, data leaks, and policy bypasses.
Release ArtifactA versioned record of the model, prompt, tools, policies, retrieval index, eval results, and deployment metadata used in a release.
ResourceIn MCP and related systems, a context object or data surface exposed for reading by a host or model application, distinct from a callable tool.
Retrieval PoisoningA failure or attack where misleading, malicious, stale, or unauthorized content enters the retrieval corpus and influences model outputs.
SandboxAn isolated execution environment for risky work such as code execution, shell commands, browser automation, file mutation, or data queries, with limits on filesystem, network, secrets, and resources.
Semantic SearchSearch based on meaning or similarity rather than exact keyword matching, usually powered by embeddings and vector indexes.
Side EffectA change caused outside the model context, such as writing a file, sending an email, updating a ticket, running code, querying a database, charging a card, or calling an external API.
SLO / SLAService-level objective / agreement. An SLO is an internal target; an SLA is a commitment that may carry external or contractual consequences.
SnowflakeEnterprise data cloud and AI platform for governed warehouse data, SQL analytics, Cortex Analyst text-to-SQL, Cortex Search retrieval, Cortex Agents, UDF/stored-procedure tools, and Streamlit-style internal data apps. In this playbook it is a business data + governance + AI platform that agents can use as a trusted substrate.
Source Maintenance DisciplineThe practice of separating stable architecture guidance from volatile market claims and requiring source URLs, owners, and last-checked dates for claims likely to drift.
Tenant BoundaryThe isolation line between customers, teams, users, or environments that prevents data, memory, permissions, and traces from crossing improperly.
Tenant-Scoped AccessAccess control that restricts retrieval, memory, tools, traces, and artifacts to the tenant, workspace, environment, or organization authorized for the current user/run.
Think → Act → ObserveThe inner agent loop where the model decides what to do, calls a tool or produces an action, observes the result, and continues until a stop condition.
Tool InterfaceThe typed boundary presented to the agent for calling an action, including name, description, parameters, output schema, permissions, and error behavior.
Tool SchemaThe structured contract for a tool, including name, description, input parameters, output shape, validation rules, and permission requirements.
Trajectory EvalAn evaluation of the full path an agent takes: reasoning steps, retrieval, tool calls, intermediate states, errors, and final answer.
Vector DatabaseA database or search layer optimized for storing embeddings and retrieving nearest-neighbor matches for semantic search.
Virtual KeyA gateway-issued key used to control model access, budgets, rate limits, team/user attribution, and spend tracking without exposing raw provider credentials to every application.
Workflow FallbackA predefined alternative path when a model, tool, retrieval source, or agent step fails or falls below confidence thresholds.
